# CliProxyAuthSweeper 开发文档

## 1. 目标

实现失效授权文件自动清理工具，满足以下约束：

- 默认运行即删除失效文件（删除模式）
- 提供观察模式（仅检测，不删除）
- 仅使用环境变量配置，不使用命令行参数
- 不创建业务状态文件
- 不依赖 `jq`

## 2. API 依据

参考：`https://help.router-for.me/cn/management/api`

使用端点：

- `GET /v0/management/usage`
- `GET /v0/management/auth-files`
- `DELETE /v0/management/auth-files?name=<file.json>`

认证头：

- `Authorization: Bearer <management_key>`
- `X-Management-Key: <management_key>`

关键字段：

- Usage 事件：`.usage.apis.*.models.*.details[]`
- 事件字段：`timestamp`、`auth_index`、`failed`
- 授权文件字段：`files[].id`、`files[].name`、`files[].source`、`files[].runtime_only`

## 3. 判定规则

### 3.1 连续失败判定

以 `auth_index` 为维度，按 `timestamp` 升序计算：

1. `failed=true`，连续失败计数 `+1`
2. `failed=false`，连续失败计数归零
3. 最大连续失败次数 `>= THRESHOLD` 时，判定该授权失效

### 3.2 统计范围

- 每次运行均按 Usage 全量数据统计
- 不使用滑动窗口，不进行跨运行增量裁剪

### 3.3 授权文件匹配

匹配顺序：

1. `auth_index == files[].auth_index`（主匹配，兼容 `files[].authIndex`）
2. `auth_index == files[].id`（回退，兼容 `id` 去掉 `.json`）
3. `auth_index == files[].name`（回退）
4. `auth_index == strip_suffix(files[].name, ".json")`（回退）

若匹配到多个文件，视为歧义，跳过并记录。
若未匹配到任何文件，视为历史残留（通常已删除），静默忽略且不在报告中体现。

### 3.4 删除过滤条件

只有满足以下条件才进入删除候选：

- `source == "file"`
- `runtime_only != true`
- `name` 以 `.json` 结尾

## 4. 运行模式

- 删除模式：`RUN_MODE=delete`（默认）
- 观察模式：`RUN_MODE=observe`

行为：

- 删除模式：调用 `DELETE /auth-files?name=...`
- 观察模式：只输出候选，不删除

## 5. 配置方式（环境变量）

- `MANAGEMENT_KEY`：必填
- `BASE_URL`：默认 `http://localhost:8317/v0/management`
- `THRESHOLD`：默认 `3`
- `RUN_MODE`：默认 `delete`
- `ALLOW_NAME_FALLBACK`：默认 `1`
- `TIMEOUT`：默认 `10`
- `INSECURE`：默认 `0`
- `VERBOSE`：默认 `0`
- `PYTHON_BIN`：默认 `python3`（回退 `python`）

## 6. 脚本实现要点

技术栈：

- `bash + curl + python3`

实现细节：

- 不使用状态文件、日志文件、报告文件
- 全流程通过内存变量和标准输出传递数据
- 使用 Python 内置 JSON 处理，避免 `jq` 依赖
- 支持 `429/5xx` 指数退避重试（1s/2s/4s）
- 删除逐条执行，单条失败不阻断后续

## 7. 输出设计

默认终端输出：

- 运行模式、窗口模式
- 事件总数、窗口内事件数
- 失效授权数、候选删除数、跳过数
- 删除成功数、删除失败数

`VERBOSE=1` 时可输出完整 JSON 报告（打印到终端，不写文件）。

## 8. 异常与退出码

- 参数或环境变量非法：退出码 `1`
- API 调用失败（非 2xx）：退出码 `1`
- 删除模式下存在删除失败：退出码 `2`
- 其余成功路径：退出码 `0`

## 9. 安全建议

- 首次建议先使用 `RUN_MODE=observe` 验证候选
- 生产再切换默认删除模式
- 对歧义匹配（同一 `auth_index` 命中多个文件）保持跳过策略，避免误删
